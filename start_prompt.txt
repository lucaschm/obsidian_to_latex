Schreibe ein Programm, welches aus Obsidian Notizen Latex Dateien erstellt. Dem Programm wird ein Pfad zum Wurzelverzeichnis gegeben. Nun gibt es für jede Latexdatei einen Unterordner. Die relevanten Ordner haben nur eine Ziffer als Namen (1-9). Alle anderen Ordner sind irrelevant und werden ignoriert. In diesen Ordnern befinden sich Obsidian-Markdown Datein. Unterordner werden ignoriert. Es werden nur Dateien berücksichtig, die ein Kapitelpräfix im Namen tragen (Beispiele: "1", "2.1", "6.12", "3.2.1"). Das Kapitelpräfix ist entscheidend, für die Sortierung der Texte. Der Inhalt der Dateien eines Ordners werden in die entsprechende Latexdatei geschrieben. Die Latex-Zieldatei hat den selben Namen wie der Ordner. Beim Einfügen des Inhalts, sind Dateinamen mit Abzug vom Kapitelpräfix als section gesetzt. Unter dieser section wird dann der Inhalt der Datei eingefügt. Beim kopieren des Inhalts einer Datei muss darauf geachtet werden, dass der YAML-Header nicht mit eingefügt wird. Außerdem wird der Notizenabschnitt auch weggelassen. Dieser hat die Markdown-Überschrift "Notes". Alles unter dieser Überschrift, wird nicht in die Latex-Datei geschrieben. Innerhalb einer Datei kann mit einem Makro auf eine andere Datei verwiesen werden. Dabei wird folgende Obsidiansyntax verwendet: ![[Dateiname]]. An der Stelle dieses Makros muss der Inhalt der Datei eingefügt werden. Die Datei muss im gesamten Wurzelverzeichnis gesucht werden. In dieser Datei können ebenfalls Makros sein. Ein solches Makro kann auch auf einen Teilbereich verweisen: ![[Dateiname#Überschrift]]. Dann wird nur der Inhalt vom Kapitel "Überschrift" dort eingefügt. Wenn das Makro auf ein Bild verweist, wird dafür \includegraphics[width=\textwidth]{Dateiname} eingesetzt. Ansonsten ist es noch wüschenswert, dass Markdown Syntax in Latex Syntax umgewandelt wird (Beispiel: Tabellen, Auflistungen)

Schreibe das Programm so, dass es sehr modular und erweiterbar ist. Die Grundfunktionalität ist das kopieren der Inhalte. Komplexere Funktionalitäten wie die Textersetzung durch Makros und die Konvertierung der Markdown syntax müssen leicht austauschbar sein. Das Programm muss auf Englisch geschrieben sein. Auch keine deutschen Kommentare.
